# Bevia.co — Build Spec

> **Platform:** Lovable (Gemini AI) + Supabase
> **Domain:** bevia.co
> **Aesthetic:** NeuroverseOS accent colors on white/light background
> **Approach:** Mobile-first responsive. Shell first, then tools one at a time.
> **Open-source strategy:** Bevia is closed-source (commercial). NeuroverseOS governance engine is open-source. Bevia drives discovery of NeuroverseOS.

---

## Branding & Attribution

Bevia is a commercial product. NeuroverseOS is the open-source engine underneath. Users don't need to know the internals — they just see results. Developers who are curious can find the engine.

### Required "Powered by" placement

1. **Footer (every page):** `"Powered by NeuroverseOS governance engine"` — links to https://github.com/NeuroverseOS/Neuroverseos-governance
2. **Result cards:** Small `"⚡ NeuroverseOS"` watermark in bottom-right of every output card — links to the repo
3. **Share cards/images:** When a result is shared as an image or public link, include `"bevia.co · powered by NeuroverseOS"` at the bottom
4. **Landing page:** One section: "Built on NeuroverseOS" — brief explanation that the governance engine is open-source, auditable, deterministic. Link to GitHub repo. This builds trust with technical users and drives open-source discovery.
5. **Align verdict reports (PDF):** Footer on every page: `"Generated by Bevia · NeuroverseOS governance engine · [repo link]"`

### What this achieves

- Bevia users who are developers discover NeuroverseOS → stars, contributors, adoption
- NeuroverseOS users who need a product discover Bevia → paying customers
- "Open-source engine" builds trust for Align (enterprise buyers want auditable, not black-box)
- The governance engine gets real-world usage proof ("powers Bevia with X thousand checks/month")

## Engine Rule: Hybrid Everything

**Every Bevia engine uses a two-pass hybrid architecture. No exceptions.**

Behavior is not word-specific. Someone might say "user" instead of "customer," or "client," or write in a different language entirely. Keywords miss conceptual meaning. AI alone is inconsistent. Hybrid gets both.

```
Pass 1: Deterministic signal detection (instant, free)
  → Structural patterns: punctuation, formatting, formality cues, red line flags
  → These are SIGNALS, not verdicts. They feed into Pass 2.

Pass 2: AI conceptual analysis (Gemini Flash, ~$0.01)
  → Reads meaning, intent, behavioral alignment, cultural context
  → Informed by Pass 1 signals for better accuracy
  → Produces the actual verdict/translation/perspective
```

**Merge rule:** Deterministic hard blocks (red lines) always win. For everything else, AI's conceptual understanding takes priority. Both sources are tagged in output for transparency.

**Why this matters for margin:** Pass 1 is free. Pass 2 is ~$0.01 (Gemini Flash). Total cost per action: ~$0.01. On a $0.05 credit, that's **80% margin**. Still double the 40% target.

**Language note:** The AI pass handles multilingual input naturally. A strategy doc in English checked against a proposal in Spanish — the AI understands both. Keywords never could.

| Color | Hex | Usage |

### Typography

| Element | Font | Size | Weight |
|---------|------|------|--------|
| Logo/Brand | `'Fira Code', monospace` | 20px | 700 |
| Headings | `'Inter', sans-serif` | 18-24px | 600 |
| Body | `'Inter', sans-serif` | 14-16px | 400 |

### Shared Components

All tools use the same component kit:

- **Cards** — white bg, `1px` border `#e2e8f0`, `8px` border-radius, subtle shadow
- **Verdict badges** — pill-shaped, color-coded (green/yellow/red), outlined on light bg
- **Buttons** — primary (purple fill, white text), secondary (outlined), ghost (text only)
- **Credit indicator** — always visible in header, shows balance, pulses on deduction
- **Tool nav** — tab bar (mobile bottom) or sidebar (desktop left)
- **Output cards** — the universal result container. Each tool fills it differently but the frame is the same: header + body + actions bar

### Auth (Supabase)

- Email + password signup
- Google OAuth
- No free tier without account — $2.99 minimum to start (buys 50 credits)
- Guest mode: can SEE the tools but hitting "Go" prompts signup + credit purchase

### Credits System (Supabase table)

```sql
-- users table (Supabase auth handles this)
-- credit_balances
create table credit_balances (
  user_id uuid references auth.users primary key,
  balance integer default 0,
  lifetime_purchased integer default 0,
  lifetime_spent integer default 0
);

-- credit_transactions
create table credit_transactions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users,
  amount integer,           -- positive = purchase, negative = spend
  action text,              -- 'purchase', 'unsaid_translate', 'align_check', etc.
  tool text,                -- 'unsaid', 'align', 'arena', 'split', 'tribe'
  metadata jsonb,           -- tool-specific details
  created_at timestamptz default now()
);
```

## What All Tools Share (the building rules)

1. **Same card frame** — every result is a card with header, content, action bar
2. **Same action bar** — Copy, Share, Save always present. Tool-specific actions added alongside.
3. **Same credit deduction UX** — header credit count pulses and decrements on use
4. **Same share system** — every output card generates a public link + screenshot image
5. **Same input pattern** — text input or file upload, always centered, always obvious
6. **Same empty state** — when no results yet, show a friendly one-liner + the input prompt
7. **Same loading state** — skeleton card with accent color pulse while AI processes
8. **Same error state** — "Something went wrong. You weren't charged." + retry button

What makes each tool **different:**
1. **Accent color** — tints header, buttons, active states
2. **Icon** — unique per tool, visible in nav and card headers
3. **Card content layout** — each tool fills the universal card differently
4. **Input type** — Unsaid = text box, Align = file upload, Split = proposal form, etc.
